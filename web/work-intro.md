## Slide 1 — 封面

**Eyebrow：** Matrix Team · ElectrifySZU

**标题：** 把宿舍电费查询做成一个可用的仪表盘

**副文：** 我们把校园电费查询、余额趋势、低电量预警和静态展示整合到一个开源项目里，让同学不用在断电前才想起查余额。

## Slide 2 — Problem

**Eyebrow：** Problem

**标题：** 同学真正需要的不是一个接口，而是少一点电量焦虑

**副文：** 查询入口分散、校园网环境有限、房间参数难找、余额变化不直观，这些小麻烦叠在一起，就会变成宿舍生活里的高频打扰。

**要点列表：**

| # | 小标题 | 描述 |
|---|--------|------|
| 1 | **查询要快** | 选择校区、楼栋和房间号后，直接展示当前余额、周期用电和预计可用天数。 |
| 2 | **趋势要清楚** | 用折线和柱状把余额走势、每日消耗和用电等级放在同一个视图里。 |
| 3 | **提醒要可靠** | 低电量订阅支持学校邮箱自动补全，减少遗漏和输入成本。 |

## Slide 3 — Architecture

**Eyebrow：** Architecture

**标题：** 静态页面负责体验，本地后端负责校园网查询

**副文：** 公开页面可部署到 GitHub Pages 做演示；真实查询时运行本地服务，由后端代理访问校园电费系统，前端只消费统一的 JSON API。

**流程图步骤：**

| Step | 名称 | 描述 |
|------|------|------|
| 01 | **输入宿舍信息** | 校区、楼栋、房间号、统计天数 |
| 02 | **发现 roomId** | 把可读房间号转换为电费系统参数 |
| 03 | **API 代理查询** | 由校园网环境中的服务请求内网接口 |
| 04 | **仪表盘渲染** | 余额、趋势、充值记录和预警状态同步更新 |

## Slide 4 — Features

**Eyebrow：** Features

**标题：** 我们把核心体验做成了完整闭环

**副文：** 从首页查询到单位切换、趋势探索、订阅保存和静态演示，每个环节都尽量让项目在展示和真实使用时都站得住。

**能力列表：**

- ✓ **双语静态仪表盘** — 中英文文案统一管理，GitHub Pages 可直接展示演示数据。
- ✓ **可交互趋势图** — 点击柱状图或折线点查看每日余额和用电详情，可自定义用电等级。
- ✓ **低电量邮件预警** — 订阅记录可保存，系统按阈值发送提醒，避免频繁打扰。
- ✓ **CLI 与 API 并存** — 既能命令行输出 JSON，也能启动本地网页服务和 API 代理。

## Slide 5 — Alerting

**Eyebrow：** Alerting

**标题：** 预警系统让查询从"想起来了"变成"被照看"

**副文：** 余额低于阈值时，系统每天最多发送一次提醒。对同学来说，重要的是在快没电前知道，而不是每天主动检查。

**信号指示器：**

| 级别 | 百分比 |
|------|--------|
| 电量充足 | 82% |
| 低电量 | 48% |
| 即将耗尽 | 18% |

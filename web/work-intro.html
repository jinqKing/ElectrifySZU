方案 · Solution

ElectrifySZU：一站式电费管家

我们是开发者，我们也是每天被断电困扰的深大人。所以我们就自己做了这个工具。

💡 核心价值：订阅预警 —— 不只是一次性查询，而是持续守护

  • 一个页面完成 校区选择 → 楼栋搜索 → 房间输入 → 一键查询
  • 中英文双语切换，GitHub Pages 可直接预览演示数据
  • 支持校园网真实查询 + 离线演示双模式

架构 · Architecture

静态页面负责体验，本地后端代理校园网查询

GitHub Pages 部署公开演示页；真实查询时运行本地服务，后端代理访问校园电费系统，前端只消费统一 JSON API。

Python (uv) + 原生 HTML/CSS/JS + CSV 存储 + SMTP 邮件

01
输入宿舍信息校区、楼栋（模糊搜索+自动补全）、房间号、统计天数
02
发现 roomId把可读房间号转换为电费系统内部参数
03
API 代理查询由校园网环境中的 Python 服务请求内网接口
04
仪表盘渲染余额、趋势、充值记录、预警状态、预计可用天数同步更新

版本 & 故事 · Version & Story

关于这个项目的一些问答

01 版本号是什么意思？
版本号是自然常数 e = 2.7182818… 的逐位截断，每发布一版往后取一位，收敛于 e 向 LaTeX 致敬——LaTeX 版本号收敛于 π，ElectrifySZU 收敛于 e。
寓意「不断电」无限进步。
02 项目名 ElectrifySZU 的启发？
项目名来自音乐剧 Hadestown 的歌曲 Chant ：
“And wasn‘t it electrifying
When I made the neon shine?”
Electrify，霓虹闪耀，我们要点亮深大 （环保前提下）
让同学们不再因为不知道余额而陷入黑暗。
03 为什么做 ElectrifySZU 这个项目？
开发团队一人有 NAS，一人有台式机。每个宿舍都用空调、都会开灯。
没人希望黑暗突然降临。 这是一个切实的痛点，所以自己动手做了这个工具。 我们是开发者，我们也是用户。
04 开发说明
本项目仅是对校园网公开查询接口的自动化封装。
所有操作仅是代替用户在校园网环境下发起请求，
调用的是宿舍电费系统系校园网公开的查询端点，
不涉及任何数据篡改、入侵、未授权访问或逆向工程。

项目代码完全开源（MIT 许可证），可审计、验证、自行部署。
我们既不存储用户密码，也不收集任何个人身份信息， 订阅邮箱服务仅用于发送预警及其它通知，可随时退订。

把宿舍电费查询做成一个可用的仪表盘

我们把校园电费查询、余额趋势、低电量预警和静态展示整合到一个开源项目里，让同学不用在断电前才想起查余额。

飞书项目 Wiki GitHub 仓库

Problem

同学真正需要的不是一个接口，而是少一点电量焦虑

查询入口分散、校园网环境有限、房间参数难找、余额变化不直观，这些小麻烦叠在一起，就会变成宿舍生活里的高频打扰。

1
查询要快

选择校区、楼栋和房间号后，直接展示当前余额、周期用电和预计可用天数。

2
趋势要清楚

用折线和柱状把余额走势、每日消耗和用电等级放在同一个视图里。

3
提醒要可靠

低电量订阅支持学校邮箱自动补全，减少遗漏和输入成本。

Architecture

静态页面负责体验，本地后端负责校园网查询

公开页面可部署到 GitHub Pages 做演示；真实查询时运行本地服务，由后端代理访问校园电费系统，前端只消费统一的 JSON API。

01
输入宿舍信息校区、楼栋、房间号、统计天数
02
发现 roomId把可读房间号转换为电费系统参数
03
API 代理查询由校园网环境中的服务请求内网接口
04
仪表盘渲染余额、趋势、充值记录和预警状态同步更新

Features

我们把核心体验做成了完整闭环

从首页查询到单位切换、趋势探索、订阅保存和静态演示，每个环节都尽量让项目在展示和真实使用时都站得住。

✓
双语静态仪表盘

中英文文案统一管理，GitHub Pages 可直接展示演示数据。

✓
可交互趋势图

点击柱状图或折线点查看每日余额和用电详情，可自定义用电等级。

✓
低电量邮件预警

订阅记录可保存，系统按阈值发送提醒，避免频繁打扰。

✓
CLI 与 API 并存

既能命令行输出 JSON，也能启动本地网页服务和 API 代理。

Alerting

预警系统让查询从“想起来”变成“被照看”

余额低于阈值时，系统每天最多发送一次提醒。对同学来说，重要的是在快没电前知道，而不是每天主动检查。

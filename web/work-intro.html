方案 · Solution

ElectrifySZU：一站式电费管家

我们是开发者，我们也是每天被断电困扰的深大人。所以我们就自己做了这个工具。

💡 核心价值：订阅预警 —— 不只是一次性查询，而是持续守护

  • 一个页面完成 校区选择 → 楼栋搜索 → 房间输入 → 一键查询
  • 中英文双语切换，GitHub Pages 可直接预览演示数据
  • 支持校园网真实查询 + 离线演示双模式

架构 · Architecture

静态页面负责体验，本地后端代理校园网查询

GitHub Pages 部署公开演示页；真实查询时运行本地服务，后端代理访问校园电费系统，前端只消费统一 JSON API。

Python (uv) + 原生 HTML/CSS/JS + CSV 存储 + SMTP 邮件

01
输入宿舍信息校区、楼栋（模糊搜索+自动补全）、房间号、统计天数
02
发现 roomId把可读房间号转换为电费系统内部参数
03
API 代理查询由校园网环境中的 Python 服务请求内网接口
04
仪表盘渲染余额、趋势、充值记录、预警状态、预计可用天数同步更新

功能 · Features

从查询到预警，完整闭环

每个特性都对应一个真实痛点，且经过多轮打磨。

⚡ 查得到 · 看得懂 · 守得住 —— 三大维度覆盖全流程

1
双语仪表盘

中英文 i18n 数据抽离，GitHub Pages 可展示；指标卡可点击切换单位（kWh/元）

2
可交互趋势图

柱状图+折线图双视图，hover 查看每日详情；可自定义用电等级阈值（低/中/高）

3
模糊楼栋搜索

输入关键词即时匹配，自动补全 campus+buildingId，有反馈提示

4
CLI + API 并存

uv run python -m src.cli json 命令行 + 本地 Web 服务双入口

5
响应式适配

移动端布局优化，QR code 可滚动

核心功能 · Killer Feature

从“想起来了才查”变成“系统替你照看”

这是 ElectrifySZU 最区别于其他查询工具的能力——不是一次性查询，而是持续守护。

  • 📧 邮箱自动补全 @szu.edu.cn
  • 🔔 双类型订阅：低电量预警 + 每日电费报告
  • ✉️ 验证确认后激活，badge 显示状态
  • 🔄 仪表盘内 toggle 开关，一键退订
  • ⏰ 低于阈值时每天最多发一封预警邮件
  • 🧪 FORCE_SEND_ALERT 测试模式方便演示
